# Auto Weekly Newsletter System — Setup Skill

You are helping a user build an automated weekly newsletter system from scratch. This system:
- Fetches news from RSS feeds weekly via a Python script
- Uses Claude AI to filter, group, and summarize relevant articles
- Generates a beautiful HTML newsletter website hosted on GitHub Pages
- Allows password-protected browser editing + one-click publish via Cloudflare Worker
- Auto-pushes updated HTML and source status Excel to GitHub after each run

Work through the steps below **in order**. At each step, ask the user for required inputs before proceeding. Do not skip steps.

---

## STEP 0 — Gather Configuration

Ask the user for the following. Collect all answers before proceeding.

1. **Newsletter topic** — What region/industry/theme? (e.g. "Mexico fintech and macroeconomic news")
2. **RSS source URLs** — A list of RSS feed URLs relevant to their topic. They can provide as few as 3 or as many as 30+. For each source ask for: URL + a short name + category (e.g. Macro & Policy / Financial Players / International Impact — or they can define their own categories).
3. **GitHub username** — Their GitHub account username.
4. **GitHub repo name** — Name for the new repository (e.g. `brazil-newsletter`).
5. **Editor password** — The password they want to use to unlock the browser editor. Must be something they'll remember. Compute its SHA-256 hash immediately:
   ```bash
   echo -n "THEIR_PASSWORD" | sha256sum
   ```
   Save both the plaintext password (for env vars) and the hash (for HTML + Cloudflare).
6. **Newsletter title** — What to display as the page title (e.g. "Brazil Weekly Newsletter").
7. **Anthropic API key** — Their `ANTHROPIC_API_KEY`. If they use a company proxy, also ask for `ANTHROPIC_BASE_URL`.
8. **Cloudflare account** — Confirm they have one (free tier is enough). They can create one at cloudflare.com.

## STEP 3 — Create the Cloudflare Worker (Publish Endpoint)

This worker handles password-verified publishing: it verifies the editor password, then writes updated HTML to GitHub via the GitHub API.

### 3a. Create the Worker

1. Go to [cloudflare.com](https://cloudflare.com) → Log in → **Workers & Pages** → **Create** → **Create Worker**
2. Give it a name (e.g. `newsletter-save`)
3. Click **Deploy** (deploys the default hello-world)
4. Click **Edit code** and **replace ALL content** with the code below:

```javascript
const CORS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'Content-Type',
  'Access-Control-Allow-Methods': 'POST, OPTIONS'
};

export default {
  async fetch(request, env) {
    if (request.method === 'OPTIONS') {
      return new Response(null, { status: 204, headers: CORS });
    }
    if (request.method !== 'POST') {
      return new Response('Method Not Allowed', { status: 405, headers: CORS });
    }

    let body;
    try { body = await request.json(); }
    catch (e) { return new Response('Invalid JSON', { status: 400, headers: CORS }); }

    const { password, html } = body;
    if (!password || !html) {
      return new Response('Missing password or html', { status: 400, headers: CORS });
    }

    // Verify password via SHA-256
    const hashBuffer = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(password));
    const hashHex = Array.from(new Uint8Array(hashBuffer)).map(b => b.toString(16).padStart(2,'0')).join('');
    if (hashHex !== env.SAVE_PASSWORD_HASH) {
      return new Response('Unauthorized', { status: 401, headers: CORS });
    }

    const owner = env.GITHUB_OWNER;
    const repo  = env.GITHUB_REPO;
    const token = env.GITHUB_TOKEN;
    const path  = 'index.html';

    // Get current file SHA (required for GitHub update API)
    const shaRes = await fetch(`https://api.github.com/repos/${owner}/${repo}/contents/${path}`, {
      headers: { 'Authorization': `token ${token}`, 'User-Agent': 'cloudflare-worker' }
    });
    if (!shaRes.ok) {
      return new Response('Failed to get file SHA: ' + await shaRes.text(), { status: 500, headers: CORS });
    }
    const sha = (await shaRes.json()).sha;

    // Base64-encode HTML (handle Unicode)
    const bytes = new TextEncoder().encode(html);
    let binary = '';
    bytes.forEach(b => binary += String.fromCharCode(b));
    const content = btoa(binary);

    // Push updated HTML to GitHub
    const updateRes = await fetch(`https://api.github.com/repos/${owner}/${repo}/contents/${path}`, {
      method: 'PUT',
      headers: {
        'Authorization': `token ${token}`,
        'User-Agent': 'cloudflare-worker',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ message: 'Newsletter updated via editor', content, sha })
    });

    if (!updateRes.ok) {
      return new Response('GitHub API error: ' + await updateRes.text(), { status: 500, headers: CORS });
    }

    return new Response(JSON.stringify({ ok: true }), {
      status: 200,
      headers: { ...CORS, 'Content-Type': 'application/json' }
    });
  }
};
```

5. Click **Deploy** → note the Worker URL (e.g. `https://newsletter-save.YOUR_SUBDOMAIN.workers.dev`)

| `GITHUB_REPO`      | their repo name |
| `GITHUB_TOKEN`     | the Personal Access Token from Step 1 |
| `SAVE_PASSWORD_HASH` | the SHA-256 hash of their editor password (computed in Step 0) |

## TROUBLESHOOTING

| Symptom | Cause | Fix |
|---------|-------|-----|
| `git push` fails with "Could not connect" | Git not using system proxy | `git config --global http.proxy http://localhost:PORT` |
| Password always wrong | EDITOR_HASH mismatch | Recompute: `echo -n "password" \| sha256sum` and update in HTML |
| Buttons don't respond | JS syntax error in script block | Check browser Console for red errors; common cause: literal newline in string |
| Source Report 404 | xlsx not in git | Check `.gitignore` has `!source_status.xlsx` exception |
| Publish fails with 401 | SAVE_PASSWORD_HASH mismatch | Must match SHA-256 of same password as EDITOR_HASH |
| Claude API 401 | Wrong base URL | Set `ANTHROPIC_BASE_URL` to company proxy if applicable |
