系统概述

本系统实现了全自动的每周新闻简报生成与发布。每周只需运行一次 Python 脚本，从抓取新闻到更新网站，全部自动完成。

📡
自动抓取新闻

从 30+ RSS 信息源批量获取过去 7 天的新闻文章，覆盖多个媒体渠道。

🤖
AI 筛选与摘要

Claude AI 自动过滤不相关内容、对同一事件去重合并，并生成精炼的中英文摘要。

🌐
自动生成并发布网站

生成精美 HTML 简报页面，自动推送至 GitHub，网站约 1 分钟内完成更新。

📊
来源状态追踪

自动生成 Excel 报告，记录每个 RSS 源的抓取状态（成功/失败/文章数量）。

✏️
在线编辑与发布

无需重新运行脚本，直接在浏览器中对内容进行微调后一键发布。

// Part 02

网站功能说明

生成的网站不只是一个静态页面——解锁编辑器后，你可以直接在浏览器中完成多种操作。

🔓 解锁编辑器

点击页面右下角的 🔒 锁图标，输入编辑密码，即可进入编辑模式。编辑器工具栏将出现在页面底部。

🖊️
编辑新闻标题与正文

点击任意新闻卡片上的编辑按钮，修改标题或摘要内容。

🗑️
删除不需要的新闻

点击卡片上的删除按钮，移除不需要展示的新闻条目。

⠿
拖拽重新排序

拖动卡片左侧的拖拽手柄，调整同一分类内新闻的排列顺序。

⬆️
一键发布到网站

完成编辑后，点击 Publish 按钮，修改内容将实时更新到公开网站。

📄
下载 Word 文档

点击 ⬇ Word 按钮，下载包含本期全部新闻标题（第一页）和标题+正文（第二页）的 .doc 文档，可直接用于内部分享。

📊
下载 Source Report

点击 ⬇ Source Report 按钮，下载本次运行的 Excel 报告，查看各 RSS 信息源的抓取状态、文章数量及失败原因。

💡 提示： 发布（Publish）操作会直接更新 GitHub 上的源文件，所有人刷新页面后即可看到最新内容。编辑内容不需要重新运行 Python 脚本，也不需要任何技术操作。

架构与服务角色

整个系统由 5 个组件协同工作，所有云服务均为免费层。

服务 / 组件 费用 在系统中的角色
🐍 Python 脚本 本地免费 核心引擎——每周手动运行一次，完成抓取→AI处理→生成网页→推送发布的全流程
🤖 Claude AI API 按量 智能处理新闻——判断相关性、去重合并、重写标题、生成摘要。每次运行调用约数百次
🐙 GitHub 免费 内容仓库——存储 index.html 和 Excel 文件；GitHub Pages 将其自动发布为公开网站
☁️ Cloudflare Worker 免费 浏览器编辑的"保存接口"——验证密码后，代替浏览器调用 GitHub API 更新网页内容
🌐 访问者浏览器 — 读取并展示网站内容；编辑者解锁后还可直接在浏览器中编辑并通过 Cloudflare 发布

使用 Claude Skill 快速复现

本项目内置了一个 Claude Code Skill，你的同事只需完成以下几步，即可让 AI 一步步引导他搭建属于自己的新闻简报系统。

/setup-newsletter
在 Claude Code 中输入此命令，AI 将引导你完成整个系统的搭建。
  • 1 复制 Skill 文件 — 将本仓库中 skills/setup-newsletter.md 复制到你新项目的 .claude/commands/ 目录下
  • 2 打开 Claude Code — 在你的项目目录中启动 Claude Code（命令行输入 claude）
  • 3 运行 Skill — 在 Claude Code 中输入 /setup-newsletter，AI 将开始引导你
  • 4 提供你的配置信息 — AI 会逐一询问：你的新闻主题、RSS 来源链接、GitHub 账号、想要使用的密码、API Key 等
  • 5 AI 自动生成代码 — AI 将根据你提供的信息，自动生成定制化的 Python 脚本，并引导你完成 GitHub 和 Cloudflare 的配置
  • 6 运行脚本，验证结果 — 按 AI 指引运行 Python 脚本，约 2-5 分钟后你的新闻网站将上线

需要提前准备的内容

  • 你关注领域的 RSS 新闻源链接（至少 3-5 个，越多越好）
  • 一个 GitHub 账号（免费注册）
  • 一个 Cloudflare 账号（免费注册：cloudflare.com）
  • 你的 Anthropic API Key（或公司内部的 Claude API 代理地址）

常见问题

⚠ git push 报网络错误
需要为 Git 配置系统代理。在终端运行：git config --global http.proxy http://localhost:端口号（端口号从 Windows 代理设置中查看）
⚠ 密码输入后提示错误
密码 Hash 设置有误。重新计算并更新：echo -n "你的密码" | sha256sum，将结果更新到 HTML 文件的 EDITOR_HASH
⚠ 按钮点击没有任何反应
JavaScript 代码有语法错误。按 F12 打开开发者工具 → Console 标签，查看红色报错信息，找到对应行号修复
⚠ Publish 后报 401 错误
Cloudflare Worker 中的 SAVE_PASSWORD_HASH 与 HTML 中的 EDITOR_HASH 不一致，必须是同一密码的 Hash
⚠ Source Report 下载 404
Excel 文件未推送到 GitHub。检查 .gitignore 是否有 !source_status.xlsx 例外规则，然后 git add source_status.xlsx
⚠ Claude API 返回 401
公司内部 API 需要设置代理地址。确认 ANTHROPIC_BASE_URL 环境变量已正确设置为公司的内部代理 URL
