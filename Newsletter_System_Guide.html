Part 1

系统概述

本系统实现了一套全自动化的每周新闻简报生成与发布流程。每次运行 Python 脚本，系统将自动完成以下全部工作：

📰

自动抓取新闻

从多个 RSS 来源批量获取过去一周的文章，支持 30+ 信息源同时抓取。

🤖

AI 筛选与摘要

调用 Claude API 对文章进行相关性过滤、去重分组、摘要生成，仅保留高价值内容。

🌐

自动生成网站

生成精美 HTML 简报页面并自动推送到 GitHub Pages，一键发布为公开网站。

✏️

在线编辑与发布

通过密码解锁浏览器编辑器，可对内容进行调整后一键发布，无需本地操作。

📊

来源状态追踪

生成 Excel 报告记录各 RSS 源的抓取状态，可直接从网站下载查看。

💡 整个系统的所有云服务均为免费层：GitHub 免费账户 + Cloudflare Workers 免费层（100k 请求/天）。唯一有成本的是 Claude API 调用（每次运行约消耗几千 tokens）。
Part 2

架构与服务角色

系统共涉及 5 个服务/组件，各自职责清晰：

服务 / 组件 类型 费用 职责
本地 Python 脚本 本地执行 免费 核心逻辑：抓取 RSS → Claude 处理 → 生成 HTML + Excel → git push
Claude API AI 服务 按量计费 相关性过滤、文章分组、标题重写、内容摘要
GitHub Repository 代码/文件存储 免费 存储 index.html 和 source_status.xlsx，是内容的"唯一真实来源"
GitHub Pages 静态网站托管 免费 将 GitHub repo 中的 index.html 自动发布为公开网站
Cloudflare Worker Serverless 函数 免费 浏览器编辑后的"保存"接口：验证密码 → 调用 GitHub API 更新 index.html
⚠️ Cloudflare Worker 的作用仅限于"浏览器端编辑后发布"这一个场景。Python 脚本直接用 git 推送，不经过 Cloudflare。

Part 4

各组件详解

Python 脚本关键模块

模块/函数作用
fetch_rss()用 feedparser 解析 RSS，过滤过去 7 天文章
filter_relevant()批量调用 Claude：判断每篇文章与主题的相关性（批次处理，节省 tokens）
group_and_summarize()调用 Claude：对相关文章去重分组，生成每组的标题和摘要
generate_html()将分组结果渲染为完整 HTML 页面（含编辑功能）
generate_excel()生成 source_status.xlsx：每个 RSS 源的抓取成功/失败状态
main() 末尾shutil.copy + subprocess git push，全自动发布

HTML 页面内嵌功能

功能技术实现
密码解锁编辑器浏览器原生 crypto.subtle.digest('SHA-256')，无后端依赖
卡片拖拽排序SortableJS CDN
Word 文档下载原生 Blob(application/msword)，无 CDN 依赖
Source Report 下载<a href="source_status.xlsx" download>
发布到 GitHubfetch POST → Cloudflare Worker → GitHub API

Cloudflare Worker 接口规范

POST https://your-worker.workers.dev
Content-Type: application/json

{
  "password": "明文密码（Worker 内部计算 hash 验证）",
  "html":     "完整 HTML 字符串"
}

Response 200: { "ok": true }
Response 401: Unauthorized（密码错误）
Response 500: GitHub API 调用失败

搭建指引（Step by Step）

准备工作：确认所需账号和工具

✅ GitHub 账号（免费）

✅ Cloudflare 账号（免费）— cloudflare.com 注册

✅ Anthropic API Key — 或公司内部代理 URL + Key

✅ Python 3.8+ 已安装，pip 可用

✅ Git 已安装，已配置用户名/邮箱

创建 GitHub 仓库 + 开启 GitHub Pages

① github.com → 右上角 + → New repository

② 仓库名自定义（如 brazil-newsletter），设为 Public，不要初始化 README

③ 创建完成后：Settings → Pages → Source: Deploy from a branch → Branch: main → 文件夹: / (root) → Save

④ 创建 Personal Access Token：Settings → Developer settings → Tokens (classic) → Generate new token → 勾选 repo 权限 → 复制 token（只显示一次！）

在本地克隆仓库

git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
cd YOUR_REPO

如果 git push 报网络错误，需配置系统代理（Windows 系统找到代理端口后）：

git config --global http.proxy http://localhost:PORT
git config --global https.proxy http://localhost:PORT

计算密码 Hash

决定你的编辑器密码，然后在 Git Bash 中计算其 SHA-256：

echo -n "YOUR_PASSWORD" | sha256sum

记录输出的 64 位 hash，后续两处会用到：HTML 中的 EDITOR_HASH 和 Cloudflare 的 SAVE_PASSWORD_HASH。

⚠️ 两处 hash 必须相同（来自同一密码）否则浏览器能解锁但无法发布。

创建 Cloudflare Worker

① 登录 cloudflare.com → Workers & Pages → Create → Create Worker

② 命名（如 newsletter-save）→ Deploy

③ Edit code → 删除全部内容 → 粘贴以下代码：

const CORS = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'Content-Type',
  'Access-Control-Allow-Methods': 'POST, OPTIONS'
};

export default {
  async fetch(request, env) {
    if (request.method === 'OPTIONS')
      return new Response(null, { status: 204, headers: CORS });
    if (request.method !== 'POST')
      return new Response('Method Not Allowed', { status: 405, headers: CORS });

    let body;
    try { body = await request.json(); }
    catch (e) { return new Response('Invalid JSON', { status: 400, headers: CORS }); }

    const { password, html } = body;
    if (!password || !html)
      return new Response('Missing fields', { status: 400, headers: CORS });

    const hash = Array.from(new Uint8Array(
      await crypto.subtle.digest('SHA-256', new TextEncoder().encode(password))
    )).map(b => b.toString(16).padStart(2,'0')).join('');

    if (hash !== env.SAVE_PASSWORD_HASH)
      return new Response('Unauthorized', { status: 401, headers: CORS });

    const base = `https://api.github.com/repos/${env.GITHUB_OWNER}/${env.GITHUB_REPO}/contents/index.html`;
    const headers = { 'Authorization': `token ${env.GITHUB_TOKEN}`, 'User-Agent': 'cf-worker' };

    const shaRes = await fetch(base, { headers });
    if (!shaRes.ok) return new Response('SHA fetch failed', { status: 500, headers: CORS });
    const sha = (await shaRes.json()).sha;

    const bytes = new TextEncoder().encode(html);
    let bin = ''; bytes.forEach(b => bin += String.fromCharCode(b));

    const upRes = await fetch(base, {
      method: 'PUT', headers: { ...headers, 'Content-Type': 'application/json' },
      body: JSON.stringify({ message: 'Newsletter updated via editor', content: btoa(bin), sha })
    });

    if (!upRes.ok) return new Response('GitHub error: ' + await upRes.text(), { status: 500, headers: CORS });
    return new Response(JSON.stringify({ ok: true }), { status: 200, headers: { ...CORS, 'Content-Type': 'application/json' } });
  }
};

④ Deploy → 记录 Worker URL（格式：https://newsletter-save.YOUR_SUBDOMAIN.workers.dev）

使用 Claude Code Skill 快速复现

Skill 包含的内容

  • ✅ 完整的 Cloudflare Worker 代码（可直接粘贴）
  • ✅ Python 脚本结构指引（Claude 会根据用户提供的 RSS 源自动生成定制版）
  • ✅ 全部环境变量配置步骤
  • ✅ GitHub Pages 开启步骤
  • ✅ 密码 hash 计算与配置方法
  • ✅ 常见错误排查表
📌 Skill 不包含具体 RSS 源——这是有意为之。每位用户需要提供自己关注领域的 RSS 源，Claude 会根据这些源自动生成定制化的 Python 脚本。
Part 7

常见问题排查

现象 原因 解决方法
git push 报 "Could not connect" Git 未使用系统代理 git config --global http.proxy http://localhost:PORT
（PORT 从 Windows 代理设置中查看）
Claude API 返回 401 Key 无效，或未设置公司代理 URL 检查 ANTHROPIC_BASE_URL 是否设置为公司内部代理
密码输入后提示错误 EDITOR_HASH 与实际密码不匹配 重新计算：echo -n "密码" | sha256sum 并更新 HTML
所有按钮无反应 JS 语法错误导致脚本块崩溃 打开 F12 → Console，查看红色报错行号，修复对应代码
Publish 返回 401 SAVE_PASSWORD_HASH 与 EDITOR_HASH 不一致 确保两者使用同一密码的 SHA-256
Source Report 下载 404 xlsx 被 .gitignore 忽略 在 .gitignore 加 !source_status.xlsx，然后 git add -f source_status.xlsx
pw-modal 相关 TypeError JS 脚本在 DOM 加载前引用了页面底部的元素 将该 addEventListener 包裹在 document.addEventListener('DOMContentLoaded', ...) 中
网站显示旧内容 浏览器缓存 Ctrl+Shift+R 强制刷新
